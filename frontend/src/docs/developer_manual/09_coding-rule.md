# 09. コーディングガイドライン

新しい機能を追加・修正する際は、以下のルールを厳守してください。

---

## セキュリティルール

- **パスワードの直接操作禁止：** DBにパスワードを保存する際は、必ず `app/core/security.py` の `get_password_hash()` を通して暗号化（bcrypt）してください。平文での保存は厳禁です。
- **権限チェックの徹底：** APIエンドポイントを作成する際は、引数に必ず適切な `Depends(...)` を含めてください（`07_権限システム.md` 参照）。
- **`SECRET_KEY` の取り扱い：** 本番環境のキーは絶対にコードベース（Git）にコミットしないでください。

---

## 開発フロー

- 新テーブル追加時は `models.py` → `schemas.py` → `routers/*.py` → `main.py` の順で実装する。
- Swagger UI（`http://localhost:8000/api/v1/docs`）でエンドポイントの動作確認を行う。
- `.env` ファイルを Git にコミットしない（`.gitignore` で必ず除外）。
- 本番DBへの直接書き込みは原則禁止。必ずローカル環境で検証してからデプロイする。

---

## 新規参画者向けオンボーディングチェックリスト

- [ ] リポジトリを clone 後、`frontend` と `backend` それぞれで依存パッケージをインストールする（`npm install` / `pip install -r requirements.txt`）
- [ ] `backend/.env` を作成し、ローカル用の設定を行う（`04_ローカル開発環境のセットアップ.md` 参照）
- [ ] `python seed_data.py` を実行し、ローカルDBにモックデータを投入する
- [ ] バックエンドとフロントエンドを起動し、ブラウザからログインできるか確認する
- [ ] （推奨）生徒一覧から進捗を1件更新し、`frontend → API → backend/router → DB` の処理フローをトレースして構造を理解する
